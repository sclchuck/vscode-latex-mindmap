# Implementation Plan: 修复 undo 栈被清空和保存按钮灰色问题

[Overview]
修复 VS Code 扩展中 undo/redo 功能失效和保存按钮状态异常的问题。核心问题是：添加/删除后立刻保存导致 undo 栈被清空，以及每次操作后都 markClean 导致保存按钮一直灰色。

问题根源在于没有区分两个不同的动作：
- `documentChanged` - 通知 extension webview 内存文档已变更，只更新缓存
- `save` / `requestSave` - 用户明确按 Ctrl+S 或点击保存时才真正写文件

修复后采用以下模型：
```
添加/删除/移动/撤销/重做 -> 修改 zustand document -> isDirty: true -> post documentChanged (不保存、不 markClean)
Ctrl+S / 保存按钮 -> post save -> extension 写文件 -> webview 收 mindmap.saved -> markClean
```

无类型系统变更，仅修改现有逻辑。
