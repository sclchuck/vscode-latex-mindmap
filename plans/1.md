下面我按你列的 7 个脚本逐个检查了。核心问题主要集中在 **MindmapNode.tsx、MindmapCanvas.tsx、D3TreeWrapper.ts**。
其中有几个会直接影响你提出的需求：

1. **展开/折叠按钮在节点外部会被裁剪**
   因为 `MindmapNode` 外层节点用了 `overflow: hidden`，而按钮 `right: -14` 在节点外，所以会被裁掉。

2. **编辑时节点大小可能在 zoom 下错误变化**
   你用的是 `getBoundingClientRect()`，它会受到 CSS `scale(zoom)` 影响。
   应改成 `offsetWidth / offsetHeight`，否则缩放后进入编辑会冻结成缩放后的尺寸。

3. **非编辑时节点根据内容自动变化，但当前测量也会受 zoom 影响**
   同样是 `getBoundingClientRect()` 的问题。应改 `offsetWidth / offsetHeight`。

4. **编辑时节点固定大小是对的，但 textarea 应只在内容区隐藏溢出，不应该让整个节点 hidden**
   否则外部按钮也被裁剪。

5. **Drag preview 只有左键按住时显示，当前逻辑基本对，但建议再补一层全局 buttons 检查**
   防止鼠标在 canvas 外释放或浏览器丢失 mouseup 后状态残留。

6. **D3TreeWrapper.ts 有一个高风险问题：`new MindmapNode(...)`**
   你从 `../types` 导入的 `MindmapNode` 大概率是 TypeScript 类型，不是运行时 class。
   如果它是 interface/type，`new MindmapNode(...)` 会直接运行时报错。
   应改成普通对象 `{ data, children }`。

# 1. DragDropService.ts 检查结果

你现在的核心逻辑已经满足：

```ts
if (!this._isMouseDown || !this._isDragging) {
    this._dropPreview = null;
    return;
}
```

以及：

```ts
if (!this._isMouseDown) {
    this._dropPreview = null;
    this._isDragging = false;
    return false;
}
```

所以 **DropPreview 必须满足鼠标按下 + 已经超过拖拽阈值** 才会出现。

不过我建议稍微加强 `SetMouseDown(false)`，但不能直接清空 `_isDragging`，因为 `MindmapCanvas.handleMouseUp` 里是先：

```ts
dragDropService.SetMouseDown(false);
handleSelectionEnd(e);
```

然后 `handleSelectionEnd` 里才调用：

```ts
const moves = dragDropService.EndDrag();
```

所以如果你在 `SetMouseDown(false)` 里面立即 `_isDragging = false`，会导致 EndDrag 拿不到移动结果。

因此 **DragDropService.ts 可以保持现状**，不建议在这里大改。

只建议把 `SetMouseDown` 改成这样，释放时只清 preview，不清 dragging：

```ts
public SetMouseDown(isDown: boolean): void {
    this._isMouseDown = isDown;

    if (!isDown) {
        this._dropPreview = null;
    }

    console.log(`[DragDropService] SetMouseDown: ${isDown}`);
}
```

这样释放鼠标后，预览一定消失，但 `EndDrag()` 仍然可以正常执行。

# 3. mindmapStore.ts 检查结果

这里有两个建议。

## 问题 1：尺寸更新没有触发 relayoutVersion

你现在：

```ts
updateNodeDimensions: (nodeId, width, height) => {
  const { nodeDimensions } = get();
  const newDimensions = new Map(nodeDimensions);
  newDimensions.set(nodeId, { width, height });
  set({ nodeDimensions: newDimensions });
}
```

这会更新尺寸 map，但不一定强制布局重新计算。你的 `MindmapCanvas` 每次 render 都重新 `calculateTreeLayout`，所以大多数情况下还行。

但更稳妥是尺寸变化时递增 `relayoutVersion`，尤其是自动测量内容变化后。

建议改成：

```ts
updateNodeDimensions: (nodeId, width, height) => {
  const { nodeDimensions, relayoutVersion } = get();
  const existing = nodeDimensions.get(nodeId);

  if (
    existing &&
    Math.abs(existing.width - width) < 1 &&
    Math.abs(existing.height - height) < 1
  ) {
    return;
  }

  const newDimensions = new Map(nodeDimensions);
  newDimensions.set(nodeId, { width, height });

  set({
    nodeDimensions: newDimensions,
    relayoutVersion: relayoutVersion + 1,
  });

  console.log(`[Store] 更新节点尺寸: ${nodeId} -> ${width}x${height}`);
},
```

## 问题 2：batchUpdateNodeDimensions 也建议触发 relayoutVersion

改成：

```ts
batchUpdateNodeDimensions: (updates: Array<{ nodeId: string; width: number; height: number }>) => {
  const { nodeDimensions, relayoutVersion } = get();
  const newDimensions = new Map(nodeDimensions);

  let hasChanges = false;

  updates.forEach(({ nodeId, width, height }) => {
    const existing = newDimensions.get(nodeId);

    if (
      !existing ||
      Math.abs(existing.width - width) >= 1 ||
      Math.abs(existing.height - height) >= 1
    ) {
      newDimensions.set(nodeId, { width, height });
      hasChanges = true;
    }
  });

  if (hasChanges) {
    set({
      nodeDimensions: newDimensions,
      relayoutVersion: relayoutVersion + 1,
    });

    console.log(`[Store] 批量更新尺寸: ${updates.length} 个节点`);
  }
},
```

# 6. MindmapCanvas.tsx 建议修改

你现在已经在 `handleSelectionMove` 里做了：

```ts

const handleMouseLeave = useCallback((e: React.MouseEvent) => {

  dragDropService.SetMouseDown(false);
  dragDropService.CancelDrag();
  dragStartPosRef.current = null;
  setDropPreview(null);
  setIsPanning(false);
}, [dragDropService]);
```

然后 JSX 改成：

```tsx
onMouseLeave={handleMouseLeave}
```

不要再直接：

```tsx
onMouseLeave={handleMouseUp}
```

## 建议补一个全局 mousemove 检查

你现在只有 window mouseup。如果鼠标在外部释放，能清理。
但如果浏览器丢失 mouseup，或者鼠标移动时 `buttons` 已经是 0，也应该清理。

加这个 effect：

```ts
useEffect(() => {
  const handleWindowMouseMove = (event: MouseEvent) => {
    const isLeftButtonDown = (event.buttons & 1) === 1;

    if (!isLeftButtonDown && (dragStartPosRef.current || dragDropService.IsDragging)) {
      dragDropService.SetMouseDown(false);
      dragDropService.CancelDrag();
      dragStartPosRef.current = null;
      setDropPreview(null);
    }
  };

  window.addEventListener('mousemove', handleWindowMouseMove);
  return () => window.removeEventListener('mousemove', handleWindowMouseMove);
}, [dragDropService]);
```

这样可以进一步保证：

> 没按住左键时，dragpreview 一定不会显示。
